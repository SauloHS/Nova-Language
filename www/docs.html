Introduction

Getting Started with Nova

Nova is a compiled, statically typed language that uses LLVM as its backend, linking with clang to produce native executables. The syntax is familiar — if you've written in Rust, you'll feel at home immediately.

Download the compiler from source using the downloads page. See the installation guide.

Hello, World

Create a file named hello.npp:

hello.npp
#include <stdio.nh>

fn main() {
    std::printf("Hello, World!\n");
}

Compile and run:

terminal
$ n++ hello.npp
$ hello
Hello, World!

Program Structure

A Nova program is a flat list of top-level declarations — global variables, global arrays, and function definitions. Execution starts at main.

Functions must be declared before they are called. Declare helper functions above their callers.
Types

Data Types

Nova has seven primitive types. All types are checked at compile time — there are no implicit conversions.

Variables

Variables

Variables require an explicit type at declaration. They need to be initialized. Declare them with let [mut] NAME: TYPE = VALUE.

let x: i32 = 0;
let mut y: i32 = 10;
let pi: f32 = 3.14;
let mut name: string = "Nova";

Assignment

Basic and compound assignment operators are all supported (only if the variable is declared as mut):

x = 42;
x += 5;   // x = x + 5
x -= 3;   // x = x - 3
x *= 2;   // x = x * 2
x /= 4;   // x = x / 4
x++;      // x = x + 1
x--;      // x = x - 1

Scope

Variables declared outside functions are global and accessible anywhere. Variables declared inside a function or block are local and destroyed when the block ends.

Operators

Operators

Arithmetic & Comparison

OperatorMeaning
+ - * /Arithmetic: add, subtract, multiply, divide

Precedence

From highest to lowest: ! → * / % → + - → comparisons → && → ||. Use parentheses to override.

&& and || use short-circuit evaluation — the right side is not evaluated if the result is already determined by the left side.
Control Flow

Control Flow

If statements have optional parentheses around the condition.

if(x > 0) {
    std::printf("positive\n");
} else if x == 0 {
    std::printf("zero\n");
} else {
    std::printf("negative\n");
}

You can chain as many else if branches as needed. The final else is optional.

Loops

Loops

while

let mut i: i32 = 0;
while(i < 10) {
    std::printf("%d\n", i);
    i++;
}

for

Classic three-part syntax: initializer, condition, step. The loop variable can be declared inside the for or already exist.

for(let mut i: i32 = 0; i < 10; i++) {
    std::printf("%d\n", i);
}

// step can be any assignment
for(let mut i: i32 = 0; i < 100; i += 5) {
    std::printf("%d\n", i);
}
Functions

Functions

Functions are declared with fn NAME(ARG1: TYPE, ARG2: TYPE) -> RETURN_TYPE { BODY }. They support full recursion.

#include <stdio.nh>

fn add(a: i32, b: i32) -> i32 {
    return a + b;
}

fn factorial(n: i32) -> i32 {
    if(n <= 1) { return 1; }
    return n * factorial(n - 1);
}

void Functions

Functions that produce side effects but return no value omit the return type entirely.

#include <stdio.nh>

fn greet(name: string) {
    std::printf("Hello, %s\n", name);
}

fn main() {
    greet("world");
}

Calling Functions

Functions can be called as expressions (capturing the return value) or as statements (discarding it):

let result: i32 = add(3, 4);   // capture return
add(10, 20);                // discard return
greet("Nova");             // void call

Struct Support in Functions

Functions can accept structs as arguments and return structs. The compiler automatically manages the stack memory layout for copying these structures.

struct MyStruct {
    x: i32;
    y: i32;
}

fn calculate_point(a: MyStruct, b: MyStruct) -> MyStruct {
    let mut c: MyStruct = {
        x: a.x + b.x,
        y: a.y + b.y
    };
    return c;
}
Arrays

Arrays

Arrays have a fixed size specified as an integer literal at declaration time. They can be stack-allocated (local) or global.

let mut nums: i32[10] = {};                          // zeroed
let mut primes: i32[5] = {2, 3, 5, 7, 11};   // initialized
let mut coords: f32[3] = {1.0, 2.5, 3.0};

let x: i32 = primes[0];        // read → 2
primes[2] = 99;           // write
primes[i + 1] = x;       // expression index
Nova performs compile-time bounds checking for literal indices (e.g. arr[10] on a size-5 array is a compile error). Dynamic indices (arr[i]) are not checked at runtime.

Arrays of Structs

You can create arrays of user-defined struct types. The compiler correctly calculates the memory offsets for accessing fields within the array elements.

struct Point {
    x: i32;
    y: i32;
}

fn main() {
    let mut arr: Point[5] = {};
    arr[0].x = 10;
    arr[0].y = 20;
    std::printf("%d\n", arr[0].x);
}
Structs

Structs

Structs let you group related data into a named type. Struct names must start with an uppercase letter.

#include <stdio.nh>

struct Point {
    x: i32;
    y: i32;
}

fn main() {
    let point: Point = {
        x: 1,
        y: 2
    };
    std::printf("%d\n", point.x);   // 1
}

Methods

Functions can be defined inside an impl block. Inside a method, all fields are accessible via self. Field changes are written back to the instance automatically.

#include <stdio.nh>

struct Counter {
    value: i32;
}

impl Counter {
    fn increment(&mut self, n: i32) {
        self.value += n;
    }

    fn get(&self) -> i32 {
        return self.value;
    }
}

fn main() {
    let mut c: Counter = { value: 0 };
    c.increment(5);
    std::printf("%d\n", c.get());   // 5
}
Struct fields support all primitive types: i32, i64, f32, f64, char, string. Nested structs are not yet supported.
Types

Explicit Casts

Use VAR as TYPE to cast. The compiler warns when precision is lost.

let a: i32 = 7;
let d: f64 = a as f64 / 2.0;   // → 3.5

let pi: f64 = 3.14159;
let truncated: i32 = pi as i32;       // → 3

let small: i32 = 1000000;
let big: i64 = small as i64 * 1000000;

let ch: char = 'A';
let code: i32 = ch as i32;             // → 65
Type Safety

Type Safety

Nova enforces strict type rules at compile time. There are no implicit conversions between numeric types — explicit casts are required.

No implicit i32 ↔ f32 conversion

Arithmetic between i32 and f32 is a compile error. Both sides must be the same type.

let x: f32 = 3.14;
let n: i32 = 2;
// let r: f32 = x * n;    ← ERROR: cannot mix f32 and i32
let r: f32 = x * 2.0;   // OK

Literal type must match declaration

Initializing an f32 variable with an integer literal is an error — the compiler suggests the fix.

// let radius: f32 = 2;     ← ERROR: use 2.0
let radius: f32 = 2.0;  // OK

Compile-time array bounds checking

Accessing an array with a literal index that is out of range is caught at compile time.

let mut arr: i32[5] = {};
// arr[10] = 1;   ← ERROR: index 10 out of bounds (valid: 0..4)
arr[4] = 1;        // OK
arr[i] = 1;        // OK — dynamic index not checked
Standard Library

Standard Library

Nova ships with a standard library under /usr/local/lib/nova/. Headers are included with #include <name.nh>. The compiler automatically finds and compiles the corresponding .npp implementation alongside your code.

All stdlib functions live under a namespace. Use the namespace:: prefix to call them — e.g. std::printf(...).
Standard Library

stdio — printf, print & scanf

Include <stdio.nh> to get std::printf for formatted output, std::print for quick single-value output, and std::scanf for formatted input. All are implemented without libc — they use syscalls directly.

hello.npp
#include <stdio.nh>

fn main() {
    std::printf("Hello, %s!\n", "World");
    std::printf("int: %d  long: %ld  double: %f\n", 42, 9876543210, 3.14);
    std::printf("char: %c  str: %s\n", 'A', "nova");
}

std::print — type-overloaded shorthand

std::print prints a value followed by a newline. The correct overload is selected automatically based on the argument type.

#include <stdio.nh>

fn main() {
    std::print(42);           // i32
    std::print(3.14);         // f64
    std::print("hello");      // string
    std::print('Z');          // char
    std::print(9876543210);   // i64 (auto-promoted)
}

std::scanf — formatted input

std::scanf reads formatted input from stdin. It works exactly like C's scanf — same format specifiers, same pointer arguments.

#include <stdio.nh>

fn main() {
    let mut n: i32 = 0;
    std::scanf("%d", &n);
    std::printf("you entered: %d\n", n);
}
Standard Library

math — sqrt

Include <math.nh> for mathematical functions. std::sqrt is overloaded for i32, f32, and f64.

#include <stdio.nh>
#include <math.nh>

fn main() {
    std::printf("%f\n", std::sqrt(2));    // i32 overload → 1.414214
    std::printf("%f\n", std::sqrt(9.0));  // f64 overload → 3.000000
}
Standard Library

Namespaces

Header files (.nh) declare a namespace with the namespace keyword. All functions declared after it belong to that namespace and must be called with the namespace:: prefix.

mylib.nh
namespace math;

fn add(a: i32, b: i32) -> i32;
fn square(x: f64) -> f64;
main.npp
#include <stdio.nh>
#include "mylib.nh"

fn main() {
    std::printf("%d\n", math::add(3, 4));
}
Calling a namespace function without the correct prefix — or without the corresponding #include — is a compile-time error.
Functions

Function Overloading

Multiple functions can share the same name as long as their parameter types differ. The compiler resolves the correct version at call time based on the argument types.

#include <stdio.nh>

fn square(x: i32) -> i32   { return x * x; }
fn square(x: f64) -> f64   { return x * x; }

fn main() {
    std::printf("%d\n",   square(5));     // → 25 (i32)
    std::printf("%f\n", square(2.5));   // → 6.250000 (f64)
}
Headers

Header Files (.nh)

A .nh file declares the public interface of a Nova library — function signatures, struct definitions, and global constants. It contains no function bodies. The corresponding .npp file provides the implementations.

When you write #include <name.nh> (for standard libraries) or #include "name.nh" (for local libraries), the compiler reads the header, registers all declarations, but passing the .npp file to the compiler is required.

Anatomy of a header

geometry.nh
// geometry.nh — public interface for the geometry library

namespace geo;

// Struct declarations are allowed in headers
struct Point {
    x: f64;
    y: f64;
}

// Function signatures — type + name + params + semicolon, no body
fn distance(x1: f64, y1: f64, x2: f64, y2: f64) -> f64;
fn area(base: f64, height: f64) -> f64;
fn sign(n: i32) -> i32;
geometry.npp
#include <stdio.nh>
#include <math.nh>

fn distance(x1: f64, y1: f64, x2: f64, y2: f64) -> f64 {
    let dx: f64 = x2 - x1;
    let dy: f64 = y2 - y1;
    return std::sqrt(dx * dx + dy * dy);
}

fn area(base: f64, height: f64) -> f64 {
    return base * height / 2.0;
}

fn sign(n: i32) -> i32 {
    if(n > 0) { return 1; }
    if(n < 0) { return 0 - 1; }
    return 0;
}
main.npp
#include <stdio.nh>
#include "geometry.nh"

fn main() {
    let d: f64 = geo::distance(0.0, 0.0, 3.0, 4.0);
    std::printf("distance = %f\n", d);   // 5.000000

    let a: f64 = geo::area(6.0, 4.0);
    std::printf("area = %f\n", a);       // 12.000000
}

Headers

Best Practices

Always declare a namespace in every .nh file

A namespace makes it explicit where a function comes from and prevents name collisions when multiple libraries are combined. Without one, all names land in the global scope and can silently shadow each other.

// ✗ Avoid — names pollute global scope, collisions are silent
fn clamp(v: i32, lo: i32, hi: i32) -> i32;
fn lerp(a: f64, b: f64, t: f64) -> f64;

// ✓ Prefer — origin is clear, call site is unambiguous
namespace math;
fn clamp(v: i32, lo: i32, hi: i32) -> i32;
fn lerp(a: f64, b: f64, t: f64) -> f64;

With a namespace, call sites become self-documenting — math::clamp(x, 0, 100) is immediately obvious in a way that bare clamp(x, 0, 100) never is.

Declare functions above their callers

Nova resolves names top-to-bottom within a file. If function B calls function A, define A first. For mutual recursion between two files, use a header to forward-declare one of them.

Use overloading instead of name suffixes

Instead of sqrtInt, sqrtFloat, sqrtDouble, define one function name with different parameter types. The compiler picks the correct version automatically — and callers don't need to care about the underlying type.

// ✗ Avoid
fn absInt(n: i32) -> i32    { return n < 0 ? 0 - n : n; }
fn absDouble(n: f64) -> f64 { return n < 0.0 ? 0.0 - n : n; }

// ✓ Prefer
fn abs(n: i32) -> i32    { return n < 0 ? 0 - n : n; }
fn abs(n: f64) -> f64    { return n < 0.0 ? 0.0 - n : n; }

Use i64 for large integers

i32 holds values up to 2,147,483,647. For counters, byte offsets, or values that can exceed that range, use i64. Integer literals larger than that limit are automatically promoted to i64 by the compiler — but when doing arithmetic that could overflow, cast explicitly before the operation.

// ✗ Overflow — arithmetic done in i32, widened too late
let bad: i64  = 1000000 * 1000000;

// ✓ Correct — cast before the multiplication
let good: i64 = 1000000 as i64 * 1000000;

// ✓ Auto-promoted — literal exceeds i32 max
let size: i64 = 9876543210;

Prefer f64 over f32 for precision

f64 gives 15–16 significant digits; f32 gives only 6–7. Use f32 only when you specifically need 32-bit representation — for example, when interfacing with hardware or external binary formats that require it.

One concern per .npp file

Keep each .npp focused on a single topic — strings, geometry, IO. Pair it with a .nh that exposes only what callers need and hides internal helpers. This makes each module independently testable and easy to reuse across projects.

Examples

Full Examples

Fibonacci

fib.npp
#include <stdio.nh>

fn fib(n: i32) -> i32 {
    if(n <= 1) { return n; }
    return fib(n - 1) + fib(n - 2);
}

fn main() {
    for(let mut i: i32 = 0; i <= 10; i++) {
        std::printf("%d\n", fib(i));
    }
}

Sieve of Eratosthenes

sieve.npp
#include <stdio.nh>

let limit: i32 = 100000;
let mut isPrime: i32[100001] = {};

fn main() {
    for(let mut i: i32 = 2; i <= limit; i++) {
        if(isPrime[i] == 0) {
            let mut j: i32 = i * 2;
            while(j <= limit) {
                isPrime[j] = 1;
                j += i;
            }
        }
    }
    for(let mut n: i32 = 2; n <= limit; n++) {
        if(isPrime[n] == 0) {
            std::printf("%d\n", n);
        }
    }
}

FizzBuzz

fizzbuzz.npp
#include <stdio.nh>

fn main() {
    for(let mut i: i32 = 1; i <= 100; i++) {
        if(i % 15 == 0) {
            std::printf("FizzBuzz\n");
        } else if(i % 3 == 0) {
            std::printf("Fizz\n");
        } else if(i % 5 == 0) {
            std::printf("Buzz\n");
        } else {
            std::printf("%d\n", i);
        }
    }
}

Struct with Methods

rectangle.npp
#include <stdio.nh>

struct Rectangle {
    width: i32;
    height: i32;
}

impl Rectangle {
    fn area(&self) -> i32 {
        return self.width * self.height;
    }

    fn perimeter(&self) -> i32 {
        return (self.width + self.height) * 2;
    }
}

fn main() {
    let mut r: Rectangle = { width: 6, height: 4 };
    std::printf("area=%d\n", r.area());           // 24
    std::printf("perimeter=%d\n", r.perimeter()); // 20
}
