Syntax

Clean by design. Fast by default.

Nova's syntax is straightforward and readable. No hidden costs, no garbage collection pauses — every line you write maps directly to optimized native instructions.

  • Static types — catch errors at compile time
  • LLVM backend — world-class optimizer for free
  • -O2 / -O3 flags — opt into aggressive optimization
  • Familiar control flow — if, while, for like you know
  • First-class functions — recursion, parameters, overloading, void
sieve.npp
#include <stdio.nh>

let limit: i32 = 1000000;
let mut isPrime: i32[1000001] = {};

fn main() {
    for(let mut i: i32 = 2; i <= limit; i++) {
        if(isPrime[i] == 0) {
            let mut j: i32 = i * 2;
            while(j <= limit) {
                isPrime[j] = 1;
                j += i;
            }
        }
    }
    let mut n: i32 = 2;
    while(n <= limit) {
        if(isPrime[n] == 0) {
            std::printf("%d\n", n);
        }
        n++;
    }
}
Why Nova

Everything you need. Nothing you don't.

Designed from the ground up for people who want C-level performance without C-level complexity.

⚡

Compiles to Native

No virtual machine, no interpreter. Nova uses LLVM to produce optimized machine code that runs at full hardware speed.

◈

Static Typing

Types are checked at compile time. i32, f32, f64, i64, char, string — no runtime surprises.

⊞

Fixed-Size Arrays

Declare arrays with static sizes, initialize with literals, and index with arbitrary expressions. Stack-allocated and fast.

⇄

Full Control Flow

if / else if / else, while loops, and rust-style for loops. Short-circuit logical operators && and || included.

ƒ

Functions & Recursion

Define typed functions with parameters, return values, and full recursion support. void functions for side effects.

◎

GCC-Style Errors

Compiler errors include file, line, column, and source context with ANSI color — so you always know exactly what went wrong.

▣

Structs & Methods

Define custom data types with named fields and methods. Call them with instance.method() syntax — no classes needed.

⊘

Type Safety

Mixing i32 and f32 in arithmetic is a compile error. Compile-time array bounds checking on literal indices.
